SerhiiUI

A modern, self-contained UI library for Roblox script hubs - sidebar tabs, rounded cards, a topbar with window controls, 11 themes, Lucide icons, an optional key system, and opt-in config saving. One file, no build step, no asset downloads.

local SerhiiUI = loadstring(game:HttpGet("https://raw.githubusercontent.com/CoderSerg/serhiiui/refs/heads/main/src/SerhiiUI.lua"))()

local Window = SerhiiUI:CreateWindow({ Title = "My Hub", SubTitle = "v1.0" })
local Tab = Window:Tab({ Title = "Main" })

Tab:Button({ Title = "Click me", Callback = function()
    SerhiiUI:Notify({ Title = "Hi", Content = "It works!" })
end })

Inspired by the look & feel of WindUI, but original code with its own identity: a violet accent and native UICorner/UIStroke styling (no external image assets).

Installation

Executor

Load the single file at runtime - it returns the library table:

local SerhiiUI = loadstring(game:HttpGet("https://raw.githubusercontent.com/CoderSerg/serhiiui/refs/heads/main/src/SerhiiUI.lua"))()

Roblox Studio

The same file works as a ModuleScript (it returns the library).

  1. Create a ModuleScript named SerhiiUI in ReplicatedStorage and paste in the source.
  2. From a LocalScript (e.g. in StarterPlayerScripts), require it.
local SerhiiUI = require(game:GetService("ReplicatedStorage").SerhiiUI)
local Window = SerhiiUI:CreateWindow({ Title = "My Hub" })

The library auto-detects Studio and parents its GUI to PlayerGui (a normal LocalScript can't write to CoreGui). A dual loader that works in both environments:

local SerhiiUI
do
    local moduleScript = game:GetService("ReplicatedStorage"):FindFirstChild("SerhiiUI")
    if moduleScript then
        SerhiiUI = require(moduleScript)
    else
        SerhiiUI = loadstring(game:HttpGet("https://raw.githubusercontent.com/CoderSerg/serhiiui/refs/heads/main/src/SerhiiUI.lua"))()
    end
end

Executor-only features degrade gracefully in Studio / live games: named Lucide icons (need HttpGet+loadstring), config saving and key persistence (need writefile/readfile) become no-ops - the UI still builds. See Icons for the in-game fix.

    Title = "My Hub",
    SubTitle = "v1.0",
    Icon = "panels-top-left",
    Size = UDim2.fromOffset(580, 440),
    ToggleKey = Enum.KeyCode.RightControl,

local Tab = Window:Tab({ Title = "Main", Icon = "house" })

Tab:Toggle({ Title = "Feature", Default = false, Callback = function(on)
    print("feature:", on)

Tab:Slider({
    Title = "Amount",

    Step = 1,
    Callback = function(v) print(v) end,

Tabs

Tabs are optional. Create as many as you like:

local Tab = Window:Tab({ Title = "Settings", Icon = "settings" })
Tab:Button({ Title = "Do thing", Callback = function() end })

Or skip tabs entirely - every element also exists on the Window. Calling them on the window appends to an implicit full-width "Main" page and the sidebar stays hidden:

local Window = SerhiiUI:CreateWindow({ Title = "Simple" })
Window:Toggle({ Title = "No tabs needed", Callback = function(v) print(v) end })
Window:Button({ Title = "Straight on the window" })

Elements

Every element exists on both Tab and Window. Stateful ones return an object with :Set(value[, fireCallback]) and :Get(); give any of them a Flag to make them config-savable. Most accept an Icon (Lucide name).

An uppercase header that groups the elements below it.

Toggle

Tab:Toggle({ Title, Desc, Icon, Default, Flag, Callback }) → :Set / :Get

Callback(value) fires with the boolean state.

local t = Tab:Toggle({
    Title = "Enable feature",
    Default = false,
    Flag = "FeatureEnabled",
    Callback = function(on) print("feature:", on) end,
})
t:Set(true)        -- update programmatically
print(t:Get())     -- read current value

Slider

Tab:Slider({ Title, Desc, Icon, Value = {Min, Max, Default}, Step, Flag, Callback }) → :Set / :Get

Callback(number) fires as the value changes.

Tab:Slider({
    Title = "Walk speed",
    Icon = "gauge",
    Value = { Min = 16, Max = 200, Default = 16 },
    Step = 1,
    Flag = "WalkSpeed",
    Callback = function(v)
        local hum = game.Players.LocalPlayer.Character
            and game.Players.LocalPlayer.Character:FindFirstChildOfClass("Humanoid")
        if hum then hum.WalkSpeed = v end
    end,
})

    Title = "Name",
    Placeholder = "type here…",

Dropdown

Tab:Dropdown({ Title, Desc, Icon, Values, Default, Multi, Flag, Callback }) → :Set / :Get

Single-select fires Callback(value); with Multi = true it fires Callback(selectedSet) (a map of value → true). Values may be strings or tables { Title, Desc, Icon, Callback }.

Tab:Dropdown({
    Title = "Targets",
    Icon = "users",
    Values = { "Players", "NPCs", "Items" },
    Multi = true,
    Flag = "Targets",
    Callback = function(selected)
        for value in pairs(selected) do print("on:", value) end
    end,
})

Keybind

Tab:Keybind({ Title, Desc, Icon, Default, Flag, Callback }) → :Set / :Get

Default is an Enum.KeyCode or its name. Click the chip, then press a key to rebind (Backspace / Escape clears). Callback(keycode) fires when the bound key is pressed.

Tab:Keybind({
    Title = "Fly",
    Icon = "keyboard",
    Default = Enum.KeyCode.F,
    Flag = "FlyBind",
    Callback = function() print("fly pressed") end,
})

Icons

Reference Lucide icons by name on the window, tabs and most elements:

Window:Tab({ Title = "Home", Icon = "house" })
Tab:Button({ Title = "Play", Icon = "play" })

Names are resolved from a remote Lucide pack the first time they're used and cached. Direct rbxassetid:// strings pass straight through.

Custom icons (and the in-game fix)

In Studio and published games, the remote pack can't load (the client has no HttpGet/loadstring), so named icons are skipped. Register your own image assets by name - these win over the pack and never touch the network:

SerhiiUI:AddIcons({
    shield = "rbxassetid://10709810572",  -- string id
    boot   = 10709790644,                 -- number id
    glyph  = { Image = "rbxassetid://1", RectOffset = Vector2.new(0,0), RectSize = Vector2.new(48,48) },
})
SerhiiUI:CreateWindow({ Title = "Hub", Icon = "shield" })

SerhiiUI:GetIcon(name) returns { Image, RectOffset, RectSize } or nil if it can't be resolved.

Why they stay crisp: the window root is a plain Frame, never a CanvasGroup (CanvasGroups flatten their children into one texture and soften every icon and glyph). Icons use ScaleType = Fit and are only ever downscaled.

Key system

An optional gate shown before the window loads. It blocks until a valid key is entered; if the user cancels, CreateWindow returns nil.

FieldTypeDescription
TitlestringHeading on the prompt.
NotestringSub-text under the heading.
Keystring | tableA key, or a list of accepted keys.
Validatorfunction(input) → boolean; overrides Key.
SaveKeybooleanRemember a valid key (executor file API).
FolderstringFolder for the saved key.
GetKeystringURL copied by a "Get Key" button.
 
local Window = SerhiiUI:CreateWindow({
    Title = "My Hub",
    KeySystem = {
        Title = "My Hub",
        Note = "Enter your key to continue.",
        Key = { "serhii123", "vip-key" },
        SaveKey = true,
        Folder = "SerhiiUI",
        GetKey = "https://example.com/getkey",
    },
})
if not Window then return end  -- user cancelled the gate

Config saving

Opt-in, and entirely up to the script author: pass ConfigFolder to CreateWindow, and give the elements you want saved a Flag. Only flagged elements persist. Values round-trip through JSON - Color3 and Enum.KeyCode are tagged so they deserialize back to the right type.

Window.Config:Save(name)write all flagged values
Window.Config:Load(name)restore them (applies callbacks)
Window.Config:List()saved config names
Window.Config:Delete(name)remove a saved config
local Window = SerhiiUI:CreateWindow({ Title = "Hub", ConfigFolder = "MyHub/configs" })
local Tab = Window:Tab({ Title = "Main" })

Tab:Toggle({ Title = "Aimbot", Flag = "Aimbot" })   -- saved (has Flag)
Tab:Button({ Title = "Not saved" })                 -- not saved (no Flag)

Tab:Button({ Title = "Save", Callback = function() Window.Config:Save("default") end })
Tab:Button({ Title = "Load", Callback = function() Window.Config:Load("default") end })

In a published game these file APIs don't exist, so save/load become no-ops. Use DataStores on the server for real persistence.

⚠ Using this in your own game

SerhiiUI is a client-side UI library. Element callbacks run on the player's own machine, so an exploiter can run them, fake their inputs, or call your code directly. "The menu only opens for admins" is not security - gating the UI client-side only hides it.

The only safe pattern for anything privileged (admin menus, shops, give-item):

  1. The client UI just asks the server - fire a RemoteEvent.
  2. The server independently verifies the requester before acting. The server is the source of truth. Clamp and sanitize everything the client sends.
-- CLIENT: the button only requests
Tab:Button({ Title = "Kick", Callback = function()
    AdminRemote:FireServer("kick", targetPlayer)
end })

-- SERVER: the server decides and acts
AdminRemote.OnServerEvent:Connect(function(player, action, target)
    if not isAdmin(player) then return end          -- re-check EVERY request
    if action == "kick" and target:IsA("Player") then
        target:Kick("Kicked by an admin.")
    end
end)

A full, copy-pasteable client + server admin menu is in examples/admin.lua.
